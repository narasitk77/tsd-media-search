Context — กุญแจสู่ความแม่นยำ

AI วิเคราะห์จากภาพอย่างเดียวอาจผิดพลาดได้ โดยเฉพาะชื่อบุคคลและสถานที่ การใส่ Context ที่ถูกต้องช่วยให้ผลลัพธ์แม่นยำขึ้นมาก

Context ระดับโฟลเดอร์ — ใส่ในช่องขวาของ Folder URL ก่อน Fetch ใช้ได้กับทุกภาพในโฟลเดอร์นั้น
Context ระดับภาพ — เปิด Edit Modal → ช่อง "Re-analyze Context" → กด Re-analyze ข้อมูลนี้จะถูกจำไว้สำหรับภาพนั้นตลอด
ค้นข่าวอัตโนมัติ — กดปุ่ม "ค้นข่าว" ใน Edit Modal เพื่อให้ AI ค้นข่าวที่เกี่ยวข้องมาใช้ประกอบ
Cross-asset context — เมื่อ batch รัน ภาพที่ 2 เป็นต้นไปในงานเดียวกันจะได้รับชื่อบุคคล/สถานที่ที่ AI ระบุได้จากภาพก่อนหน้า โดยอัตโนมัติ
ปุ่มและฟังก์ชัน
Fetch from Mimir ดึงรายการรูปภาพจาก URL ที่ระบุ
Run AI Batch วิเคราะห์ภาพ pending ทั้งหมดด้วย AI
Push All อัปโหลด metadata ทุก "done" ขึ้น Mimir
Push by Album Push เฉพาะ album/folder ที่เลือก
Bulk Edit เลือกหลายรูป (checkbox) แล้วแก้ field พร้อมกัน
Reset to Pending รีเซตภาพกลับเป็น pending เพื่อ re-analyze
Backfill Proxy ดึง proxy image URL สำหรับภาพที่ยังไม่มี
Clear DB ลบ asset ทั้งหมดออกจาก local DB
Edit Modal — การแก้ไข metadata

กดปุ่ม ในตาราง เพื่อเปิด modal สำหรับดูและแก้ไขข้อมูลทุก field

  • Title — รูปแบบ YYYY.MM.DD_หัวข้อ_ชื่อบุคคล
  • Description — คำบรรยายสไตล์สำนักข่าว AP/Reuters
  • People / Speaker — ชื่อบุคคลในภาพ คั่นด้วย comma
  • Emotion / Mood — อารมณ์ภาพ เช่น Happy, Serious, Celebratory
  • Re-analyze Context — context เฉพาะภาพนี้ ถูกจำไว้แม้จะ re-analyze ซ้ำ
สถานะภาพ
pending รอการวิเคราะห์
processing กำลังวิเคราะห์อยู่
done วิเคราะห์แล้ว พร้อม Push
error เกิดข้อผิดพลาด — กด เพื่อ retry

Changelog & Roadmap
Roadmap — AI Accuracy Improvements

แผนปรับปรุงความแม่นยำการวิเคราะห์ภาพ (เรียงตาม impact)

High Impact Cross-asset Context Sharing
ภาพจากงานเดียวกัน (path เดียวกัน) วิเคราะห์ภาพแรกก่อน แล้วนำผล (ชื่อคน, สถานที่, ชื่องาน) เป็น context อัตโนมัติให้ภาพที่เหลือในงานนั้น — ไม่ต้องพิมพ์ context เอง ชื่อคนและสถานที่สอดคล้องกันทุกภาพ
High Impact GPS → ชื่อสถานที่ (Reverse Geocoding)
ถ้า EXIF มี GPS coordinates แปลงเป็นชื่อสถานที่จริงผ่าน OpenStreetMap Nominatim (ฟรี) ก่อนส่งให้ AI — AI จะรู้ location แน่ชัดแทนการเดา
Medium Impact Event Name Extraction จาก Path
Extract ชื่องานจาก ingest_path เป็น field แยกชัดๆ ใน prompt แทนที่จะฝังอยู่ใน path string ทั้งก้อน — AI อ่านชื่องานได้ถูกต้องขึ้น
Low Impact ส่งภาพ Hi-res แทน Thumbnail
ดาวน์โหลดไฟล์จริงจาก Mimir แทน thumbnail — อ่านป้ายชื่อ, ใบหน้า, ตัวหนังสือในภาพได้ชัดขึ้น แต่ช้าและใช้ bandwidth มาก
